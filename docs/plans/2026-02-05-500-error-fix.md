# 500 Internal Server Error 修复报告

> **状态:** 已解决

**问题:** 后端启动后返回 500 Internal Server Error on `/api/info` endpoint

**根本原因:** 环境配置问题，不是代码问题

---

## 问题分析

### 症状
- 编译时报错 `rustc 1.83.0 is not supported by the following packages`
- 服务器启动报错 `Migrate(VersionMismatch(20260126000000))`

### 根本原因

1. **Cargo MSRV 检测错误**
   - rust-toolchain.toml 指定 `nightly-2026-01-21`
   - Homebrew 安装的 rustc 1.83.0 位于 `/opt/homebrew/bin/`
   - rustup 安装的 nightly 位于 `~/.cargo/bin/`
   - PATH 优先级导致 cargo 使用错误的 rustc 进行 MSRV 检查

2. **数据库迁移版本不匹配**
   - 之前修改了 migration 文件内容（从 no-op 改为实际 ALTER TABLE）
   - 已迁移数据库的 checksum 与新文件不匹配

---

## 解决方案

### 临时修复（当前会话）

编译时显式设置 RUSTC：
```bash
RUSTC=/Users/leone/.rustup/toolchains/nightly-2026-01-21-aarch64-apple-darwin/bin/rustc \
  cargo build --bin server
```

重置开发数据库：
```bash
rm -f dev_assets/db.sqlite
```

### 永久修复建议

1. **调整 PATH 优先级**（推荐）
   在 `~/.zshrc` 或 `~/.bashrc` 中：
   ```bash
   export PATH="$HOME/.cargo/bin:$PATH"
   ```

2. **或者移除 Homebrew 的 Rust**
   ```bash
   brew uninstall rust
   ```
